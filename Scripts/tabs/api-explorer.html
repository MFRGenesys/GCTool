JSON d'exemple utilise dans les expressions ci-dessous: 
{
  "results": [
    {
      "conversationId": "c-001",
      "participants": [
        { "participantId": "p-101", "participantName": "Mobile Number, France", "purpose": "external" },
        { "participantId": "p-102", "participantName": "TEL_3639", "purpose": "ivr" }
      ],
      "queue": { "id": "q-01", "name": "Support" }
    },
    {
      "conversationId": "c-002",
      "participants": [
        { "participantId": "p-201", "participantName": "Secure_Core_Model", "purpose": "botflow" }
      ],
      "queue": { "id": "q-02", "name": "Sales" }
    }
  ],
  "pageNumber": 1,
  "pageCount": 2,
  "totalHits": 3
}
$
Role: Racine du JSON.
Exemple: $ retourne tout l'objet.
$.results
Role: Acces direct a une propriete (notation point).
Exemple: retourne le tableau des conversations.
$['results'][0]['queue']['name']
Role: Acces par notation crochet (utile si cle complexe).
Exemple: retourne "Support".
$.results[0]
Role: Index d'un tableau (0 = premier element).
Exemple: retourne la conversation c-001.
$.results[*]
Role: Wildcard sur tableau (tous les elements).
Exemple: retourne toutes les conversations.
$.results[*].conversationId
Role: Projection d'un champ sur tous les elements.
Exemple: ["c-001", "c-002"].
$.results[*].participants[*].participantId
Role: Wildcards imbriques sur plusieurs niveaux.
Exemple: ["p-101", "p-102", "p-201"].
$..participantId
Role: Recherche recursive d'une cle partout dans le JSON.
Exemple: retrouve tous les participantId meme en profondeur.
$..*
Role: Tous les descendants (recursive wildcard).
Exemple: retourne toutes les valeurs descendantes.
$.results[?(@.queue.name == 'Support')].conversationId
Role: Select particular children using logical expression (filtre logique).
Exemple: retourne uniquement les conversations de la queue Support, ici ["c-001"].
$.results[*].conversationId, $.results[*].queue.name
Role: Multi-filtres dans gcTool (separes par virgule).
Exemple: combine deux projections en une vue.
Dans cette version gcTool: slices [start:end] et unions [0,2] non pris en charge.
